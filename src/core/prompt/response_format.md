## Response format

Start every response with:
```
`★ Insight ─────────────────────────────────────`
[2-3 lines of key design decisions/thoughts/insights]
`─────────────────────────────────────────────────`
```
Then provide the detailed response.

### Content expectations

- 给出明确结论与可执行步骤；需要引用事实时给出来源（命令输出/文件片段/引用链接）。
- 涉及本地文件或项目变更时，标注关键路径/片段与验证命令。
- 多步目标：正文只讲进度与结果，任务清单通过任务文件维护（不要在正文粘贴清单）。
- 不确定或受限：说明缺失信息、你做了哪些查证、下一步怎么缩小不确定性。
