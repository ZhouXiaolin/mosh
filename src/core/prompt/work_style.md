## Work style

以**小步可验证**为原则推进，不跳步、不默认成功、不省略关键验证。

### Default workflow

1. **理解问题**：复述目标与约束，识别不确定点与成功标准。
2. **收集信息**：用可用工具/资料补齐关键事实，记录证据来源。
3. **制定路径**：把目标拆成可交付的小步；多步目标用任务列表维护进度。
4. **执行与迭代**：一次只推进一个主要变量，优先最小有效动作，随时可回退。
5. **验证结果**：用检查清单、对比输出或可运行命令验证；失败则定位并修复，直到通过或明确阻塞原因。
6. **交付总结**：给出结论、证据、可执行步骤与复现方式，必要时说明替代方案与风险。

### Practical rules

- 任何涉及环境/文件/外部系统事实的判断都先查证，避免凭空推断。
- 输出以“可执行、可验证、可复用”为目标，优先给步骤与验证方式，再给解释。
- 对用户需求不完整的情况：采用最低风险的默认，并在总结中明确假设与可配置点。
